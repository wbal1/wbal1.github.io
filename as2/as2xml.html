플래시 AS2 - XML 파일 읽기

원글 출처 (영문)   홈으로            본 페이지는 1920px 너비의 브라우저 환경에 최적화되어 있습니다.

XML 지원은 플래시 5에서 처음 추가되어, 플래시 4에서는 지원되지 않습니다. 본 튜토리얼은 플래시 무비가 Flash MX 및 후속 버전을 사용하여 어떻게 외부 XML 파일로부터 데이터를 되찾아오는지를 설명합니다.

  1. 배경

    2. 왜 파일로부터 데이터를 읽을 필요가 있을까요? "오늘의 뉴스"를 표시하는 무비가 있다고 합시다. SWF 파일 속에 뉴스 내용을 담을 수 있겠지만, 그렇게 하면, 매 번 뉴스가 바뀌면, 플래시 파일을 편집해야 합니다. 이는 매우 실용적이지 않습니다. 그리하여 이를 통해 1. (편집하고 플래시 파일을 재컴파일하는) 시간을 절약하고; 2. 당신의 앱에서 (캡슐화, 구현을 숨기는) 데이터를 분리한다는 이득을 얻을 수 있습니다.

    당신은 외부 데이터를 보내기 위해 일반 텍스트 파일을 보낼 수 있습니다- 여기(영문)을 참조하십시오. 그리고 외부의 데이터를 보내는 또 다른 방법은 쿼리 스트링(영문)을 사용하거나 FlashVars(영문)을 사용하여 데이터를 포함시키는 것입니다.

    왜 XML을 사용하나요? XML을 사용하는 것의 이득은 당신이 데이터를 더욱 우아한 서식으로 만들 수 있고; 그리고 또, XML이 요즘 광범위하게 사용됨에 따라, 잠재적으로 다른 의도를 위해 정확히 같은 XML 파일이 쓰일 수 있습니다. XML 파일이 데이터의 계층적인 구도를 나타냄에 매우 좋음에 따라, 다른 파일 형식을 요구하는 지저분한 형식을 피할 수 있습니다.

    XML을 사용하는 것의 주요 약점은 액션스크립트 코딩에서 늘어난 복잡성이고; 거기에 몇몇 플래시 5 플레이어들이 의도된 방식으로 XML 파일을 읽지 못할 수 있는 버그가 있습니다(아래에서 더 봅시다).

  2. 요구사항

    3. 이 튜토리얼을 이해하기 위해서는, 다음을 아는 것을 추천드립니다:

    • 텍스트 박스를 통해 플래시에서 변수 불러오기
    • 약간의 일반적인 플래시 액션스크립트 작성 지식 및 구문
    • 약간의 기본적인 XML 지식 (계층적 구조, 형식화)
    이 튜토리얼은 플래시 MX를 사용하지만, 플래시 MX 2004에서도 적용 가능합니다. 플래시 5에서도 사용 가능하지만, 아래 주의사항을 참조하세요:
    참조: 이 튜토리얼은 적절하게 동작하기 위해 플래시 6/MX 플러그인/플레이어를 요구합니다. XML이 플래시 5에서 지원된다 하더라도; 약간의 오래된 플래시 5 플레이어는 무비가 몇몇 XML 파일들을 정확하게 불러올 수 없는 것을 초래합니다. 이 문제는 플래시 플레이어 5.0.41 버전에서 발생하였고 5.0.42 버전에서 수정되었습니다. 위의 예제가 당신의 기기에서 동작하지 않는다면, 매크로미디어에서 최신 버전의 플래시 플레이어를 다운로드해야 합니다. 플래시 5 플레이어 버전들을 지원해야 한다면, 해결 방법을 보기 위해 이 링크(영문)를 보시기 바랍니다.

  3. XML 샘플 파일

    4. 이 튜토리얼에서 쓰일 XML 파일은 "news.xml"이라고 불립니다. 이는 여기서 다운로드할 수 있습니다. XML 파일의 구조는 아래에서 나타납니다. 플래시가 아무튼 무시할 것이기에 DTD는 사용하지 않을 것임을 알아두십시오.

    <?xml version="1.0" standalone="yes"?> <news> <header>Vacation is Good</header> <content>Scientists discovered a cure for the .......</content> <info more_info="www.permadi.com"> <author>F. Permadi</author> <comment>No comment</comment> </info> </news>

    XML 파일의 루트가 <news>라는 것을 기억하십시오. <news> 속성은 <header>, <content>, <info> 요소를 1개씩 포함합니다. <info> 요소는 "more_info" 특성을 포함하고 <comment>와 <author> 요소를 포함합니다.

    XML 편집에 관한 노트

    만일 요소의 값으로써 다음 중 하나에 해당하는 특수 문자를 사용한다면, 이름 객체를 사용해야 합니다:

    기호 이름 객체
    > &gt;
    " &quot;
    ' &apos;
    & &amp;
    < &lt;

    예를 들어, <header>를 Vacation is Good & Healthy로 대체하려면 이렇게 하십시오:

    <header>Vacation is Good &amp; Healthy</header>

    이렇게가 아니라:
    <header>Vacation is Good & Healthy</header>

    이 무비는 4개의 텍스트 상자를 가졌습니다:

    • header를 표시하는 것
    • Loading data...를 표시하는 것
    • author를 표시하는 것

    각각의 텍스트는 동적 텍스트이고, (아래와 같이 속성 탭의 "변수"에서 명시된) 변수 이름과 연결되어 있습니다.

    header 텍스트 상자는 header 변수와 연결되어 있습니다. 이 상자는 <header> 뉴스를 유지할 것입니다. 유저가 텍스트를 선택하지 못하도록 이 버튼을 눌러 비활성화하게 하십시오.

    Loading Data 텍스트 상자는 content 변수와 관련되어 있습니다. 이 상자는 <content> 뉴스를 포함할 것입니다.

    author 텍스트 상자는 author 변수와 관련되어 있습니다. 이 상자는 <author> 뉴스의 이름을 포함할 것입니다.

    지금은 <date>와 <comment>같은 다른 XML 속성을 무시할 것입니다.

  5. 플래시에서 XML 파일 불러오기

    6. 다음 코드는 XML 항목을 불러옵니다:

    var xmlData=new XML(); xmlData.ignoreWhite=true; xmlData.load("news.xml");

    첫 번째 줄은 XML 객체를 만듭니다. 다음 줄은 플래시에게 XML 요소 간에 화이트스페이스를 무시하라고 알려줍니다. 보통은, 당신은 이를 항상 true로 설정하고 싶어하게 되고, 그렇지 않으면 플래시는 XML 파일을 제대로 읽지 않을 것입니다. 마지막 줄이 실제로 XML 파일을 로드하는 코드입니다.

    이렇게 외부 파일을 로드한다면, 파일이 즉시 로드된다는 가정을 내리지 말아야 한다는 것에 주목할 가치가 있습니다. 플래시가 이 파일이 얼마나 큰 지 모르기 때문에, 무비가 계속 실행되는 동안 주로 백그라운드에서 이런 과정을 거칩니다. 그러나, 플래시는 파일이 이벤트 핸들러라고 불려지는 것에 의해 로드될 때 우리에게 알려줄 만큼 사려깊습니다. XML의 경우에는, 이벤트 핸들러는 onLoad라고 불립니다. (다시 말해서, 파일이 로드되면, 플래시는 onLoad라고 부릅니다.) 우리는 onLoad 코드를 구현할 것입니다.

    참조: 이 튜토리얼에서는, XML 파일이 SWF 파일과 같은 도메인에 위치해있어야 합니다. 다른 도메인의 XML 파일에 접근해야 한다면, XML 파일에 <cross-domain-policy>를 작성함으로서 다른 도메인 간에 접근을 허용하여 다른 도메인 간에 연결합니다.

    예시는 다음과 같습니다:

    <?xml version="1.0"?> <cross-domain-policy> <allow-access-from-domain="www.friendofpermadi.com"/> </cross-domain-policy>

    도메인 간 정책을 수행해야 하는 경우, 매크로미디어 사이트(영문, 아카이브)에서 더 많은 정보를 찾을 수 있습니다.

  6. 플래시에서 XML 파일 파싱하기

    7. 이제 파일을 로드하는 방법을 알기에, 이것으로 무언가를 하는 것을 아는 것 없이는 용도가 많지 않을 것입니다. 하지만 파일을 처리하기 전에, 파일이 로드되었다는 것을 알 필요가 있습니다. 우리는 onLoad 이벤트 핸들러를 XML 파일을 처리할 우리만의 함수로 오버라이딩할 것입니다. 우리 핸들러 함수는 processXMLData라고 불립니다. 그러면 코드는 다음과 같이 됩니다:

    function processXMLData(success) { if (success) { } else { content = "Today's news is not found"; } } var xmlData = new XML(); xmlData.ignoreWhite = true; xmlData.onLoad = processXMLData; xmlData.load("news.xml"); stop();

    processXMLData가 불러질 때까지 무비를 계속하고 싶지 않기 때문에 플래시에게 무비를 정지하라고도 말할 것입니다. (이는 로딩 후에 무비를 항성 멈춰야 한다는 것을 뜻하지 않습니다. 이는 매우 당신의 무비에 달려 있습니다. 제 간단한 예시에는, 무비가 파일이 로드되기 전까지는 아무것도 하지 않기에 무비를 정지하고 싶습니다.)

    안전하기 위해서 onLoad=processXMLData 전에 processXMLData를 넣어야 한다는 점을 기억하세요. processXMLData를 위한 코드는 조금 뒤에 쓸 것입니다; 현재로써는, 이는 아무것도 하지 않습니다. success 매개 변수는 파일이 성공적으로 로드되었는지 아닌지를 나타내는 불리언 값입니다. 파일이 발견되지 않았다면 false가 될 것입니다.

    그러나 왜 모든 nodeValue이 null이죠? 이는 XML 항목이 플래시에서 동작하는 방법입니다: nodeValue은 자식 노드로써 저장됩니다. 그래서, <header>의 값은 header 노드의 자식 노드가 됩니다. 우리는 이렇게 header의 값을 얻을 수 있습니다:

    header = this.childNodes[0].childNodes[0].childNodes[0].nodeValue;

    전에 플래시에서 XML을 전혀 써본 적이 없다면, 이 모든 게 헷갈려 보일 수 있습니다. 약간의 코드를 작성하고 속성 몇 개에 접근하는 것을 추천합니다. 디버거는 여기서 위와 같이 트리 뷰를 볼 수 있기에 매우 도움이 됩니다.

    한 번만 개념에 익숙해지면, 순차적 단계 없이 XML 파일의 모든 항목에 접근할 수 있게 됩니다. 지금은, 플래시 무비에 코드를 사용하기 충분한 정보가 있습니다.

    function processXMLData(success) { if (success) { header = this.childNodes[0].childNodes[0].childNodes[0].nodeValue; content = this.firstChild.childNodes[1].firstChild.nodeValue; } else { content = "Today's news not found"; } } var xmlData=new XML(); xmlData.ignoreWhite=true; xmlData.onLoad=processXMLData; xmlData.load("news.xml"); stop();

    그 코드 조각에 대한 노드에 접근하는 방식은 매우 일관되지는 않아 보이지만, 괜찮습니다, 목적을 보여주기 위해 이 방식으로 일을 하고 있습니다.

    쿨하군요. header와 content라고 텍스트 상자들을 이름지었기 때문에, 그들은 우리가 방금 할당한 값을 자동으로 보여줄 것입니다. 그래서 우리는 이제 <author> 노드 값을 출력하기 위한 코드만을 더해야 합니다.

    <author> 노드는 <title>보다 한 층 더 깊고 <info 노드 하에 네스팅되어 있기에, 노드에 접근하기 위한 긴 코드는:

    newsNode = this.firstChild; infoNode = newsNode.childNodes[2]; authorNode = firstChild.childNodes[1]; author = authorNode.firstChild.nodeValue;

    이제 코드를 쓰고, 그 동안, 코드를 더 깔끔하게 만들기 위해 노드에 접근하는 방법을 다시 써봅시다.

    function processXMLData(success) { if (success) { var newsNode=this.firstChild; var headerNode=newsNode.childNodes[0]; var contentNode=newsNode.childNodes[1]; var infoNode=newsNode.childNodes[2]; var authorNode=infoNode.childNodes[1]; header=headerNode.firstChild.nodeValue; content=contentNode.firstChild.nodeValue; author=authorNode.firstChild.nodeValue; } else { content = "Today's news is not found"; } } var xmlData=new XML(); xmlData.ignoreWhite=true; xmlData.onLoad=processXMLData; xmlData.load("news.xml"); stop();

    이런, 코드는 여전히 살짝 어질러져 있군요. 우리 XML 파일은 꽤 단순하지만, 만일 누군가가 노드를 바른 순서에 놔야 한다는 것을 까먹는가면 어떨까요? 만약 누군가가 <content> 다음에 <header>를 넣으면 어떨까요? 코드를 더 깨끗하게 만들고 더 많은 노드를 처리할 수 있게 합시다. 또한, 좋은 XML 연습이 될 것입니다.

    노드를 반환하도록 함수를 작성합시다.

    function findNode(node, nodeName) { if (node.nodeName==nodeName) { return node; } for (var i=0; node.childNodes && i<node.childNodes.length; i++) { var foundNode=findNode(node.childNodes[i], nodeName); if (foundNode!=null) { return foundNode; } } return null; } function getValue(node) { if (node && node.firstChild) { return node.firstChild.nodeValue; } return ""; }

    findNode 함수는 통과한 node가 특정한 nodeName을 가졌는지 확인할 것입니다. 아니면, 코드는 자식 노드를 계속 돌고 다시 확인합니다(같은 함수를 호출합니다). 결국에는, 이름 붙은 노드는 찾아지거나 아니게 됩니다. 아니라면, null이 반환됩니다. (참고: 이 함수는 XML 파일 아래에 같은 이름을 가진 노드가 여러 개 있을 경우에 올바르게 동작하지 않을 것입니다. 그에 대해서는, 어느 노드를 반환할지 명시해야 합니다— 그러나 이는 튜토리얼로 설명하기에는 너무 멀리 갑니다.)

    getValue 함수는 단순히 노드의 값을 반환합니다. 이는 firstChild.nodeValue를 치기 위한 속기입니다. 이는 단지 노드의 값을 원할 매 시간마다 지저분함을 줄여 줍니다.

    이제 이러한 변화를 포함합시다:

    function processXMLData(success) { if (success) { var rootNode=this.firstChild; var headerNode=findNode(rootNode, "header"); header=getValue(headerNode); var contentNode=findNode(rootNode, "content"); content=getValue(contentNode); var authorNode=findNode(rootNode, "author"); author=getValue(authorNode); } else { content="Today's news is not found"; } } function findNode(node, nodeName) { if (node.nodeName==nodeName) { return node; } for (var i=0; node.childNodes && i<node.childNodes.length; i++) { var foundNode=findNode(node.childNodes[i], nodeName); if (foundNode!=null) return foundNode; } return null; } function getValue(node) { if (node && node.firstChild) { return node.firstChild.nodeValue; } return ""; } var xmlData=new XML(); xmlData.ignoreWhite=true; xmlData.onLoad=processXMLData; xmlData.load("news.xml"); stop();

    우리의 예시에서 attributeName가 more_info인 node.attributes.attributeName를 사용해서 플래시에서 노드의 속성을 얻을 수 있습니다.

    function processXMLData(success) { if (success) { var rootNode=this.firstChild; var headerNode=findNode(rootNode, "header"); header=getValue(headerNode); var contentNode=findNode(rootNode, "content"); content=getValue(contentNode); var authorNode=findNode(rootNode, "author"); author=getValue(authorNode); var infoNode=findNode(rootNode, "info"); if (infoNode) { moreInfo=infoNode.attributes.more_info; } } else { content="Today's news is not found"; } } // ... // 코드의 나머지 부분은 수정되지 않았고 표시되지 않았습니다
