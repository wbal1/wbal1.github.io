Nuklear

원글 출처   홈으로            본 페이지는 1920px 너비의 브라우저 환경에 최적화되어 있습니다.

  1. 개요

    2. 이것은 ANSI C로 쓰여졌고 퍼블릭 도메인 하의 최소 상태 즉시 모드 그래픽 사용자 인터페이스입니다. 이것은 애플리케이션을 위한 단순한 내장 가능 그래픽 사용자 인터페이스이고 어떠한 종속성도 가지고 있지 않는, 기본 렌더 백엔드 혹은 OS 창 및 입력 핸들러이지만 매우 모듈적인 라이브러리를 제공하기보다는 단순한 입력 상태를 통해 접근하고 출력으로써 원시적인 도형을 설명하는 명령을 그립니다. 그래서 많은 수의 플랫폼을 추상화하고 백엔드를 렌더링하는 것을 시도하는 계층화된 라이브러리보다 이는 진짜 UI에만 집중합니다.

  2. 하이라이트

    • 그래픽 사용자 인터페이스 툴킷
    • 단일 헤더 라이브러리
    • C89(a.k.a. ANSI C 혹은 ISO C90)로 작성됨
    • 작은 코드베이스 (~1.8만 줄)
    • 휴대성, 효율성 및 단순성에 중점을 둠
    • 종속되지 않음 (필요없다면 기본 라이브러리마저)
    • 완전 사용자화 및 스킨 씌우기 가능
    • 필요 시 전면적 메모리 컨트롤로 적은 메모리 발자국을 남김
    • UTF-8 지원
    • 전역 및 숨겨진 상태 없음
    • 사용자화 가능한 라이브러리 모듈 (필요한 것만 컴파일하고 사용 가능)
    • 선택적 폰트 굽기 및 정점 버퍼 출력
    • GitHub에서 코드 사용 가능

  3. 기능

    • 절대적으로 전혀 플랫폼 종속 코드가 아님
    • 메모리 관리 조종 범위:
      • 표준 라이브러리에서 모든 것을 할당함으로써의 쉬운 사용에서
      • 라이브러리 내의 모든 바이트를 조종하기까지
    • 폰트 핸들링 조종 범위:
      • 모든 것에 자신만의 폰트를 사용하는 것부터
      • 이 라이브러리의 폰트 굽기와 핸들링 API를 사용하는 것까지
    • 도면 출력 제어 범위:
      • 도면 호환성을 가진 고수준 API용 간단한 도형에서
      • 하드웨어 접근 가능한 안티앨리어싱이 적용된 정점 버퍼 출력까지
    • 커스텀 가능한 색상과 속성의 범위:
      • 간단한 색상 테이블을 채움으로써의 간단한 색상 변화에서
      • 위젯을 꾸미기 위한 스키닝의 사용 가능성을 포함한 완전한 조종까지
    • 구부릴 수 있는 UI, 근데 다음 위젯 범위를 포함한:
      • 버튼, 체크박스, 슬라이더 등의 기본 위젯부터
      • 추상적인 콤보박스, 컨텍스트 메뉴 등의 고급 위젯까지
    • 필요한 것만을 컴파일하기 위한 컴파일 시간 설정
      • 표준 라이브러리를 사용하거나 링크하기 싫을 때 사용할 수 있는 서브셋
    • 프레임 업데이트 대신 유저 입력 시 업데이트를 위해 쉽게 수정할 수 있음

  4. 사용법

    5. 이 라이브러리는 하나의 단일 헤더에 자체적으로 포함되어 있고 헤더 전용 모드나 실행 모드에서 사용할 수 있습니다. 헤더 모드는 포함되었을 때 기본으로 사용되고 이 헤더를 다른 헤더에 포함하는 것을 허용해 실제 실행을 포함하지 않습니다.

    실행 모드는 하나의 C/CPP 파일에 #include하기 전에 전처리 메크로 NK_IMPLEMENTATION을 정의하는 것을 요구합니다.

    예:

    #define NK_IMPLEMENTATION #include "nuklear.h"

    또한 추가적으로 "추가 정의" 섹션에 열거되어 있는 기호를 실행 모드에서 헤더 밑에 추가적인 기능성을 사용하고 싶거나 라이브러리에 대한 더욱 많은 조종을 필요로 할 때 정의할 수 있습니다.

    ⚠️
    Nuklear가 포함될 때마다 같은 컴파일러 플래그를 정의하십시오. 이 매우 중요한 것은 하지 않으면 컴파일러 에러나 더욱 나쁜 스택 오류로 이어질 수 있습니다.

  5. 플래그

    6. 플래그 설명
    NK_PRIVATE 정의되면 모든 함수를 정적으로 선언해, 실행을 포함한 파일의 내부에서만 접근 가능하게 합니다.
