Nuklear

원글 출처   홈으로            본 페이지는 1920px 너비의 브라우저 환경에 최적화되어 있습니다.

  1. 개요

    2. Nuklear는 ANSI C로 쓰여졌고 퍼블릭 도메인 하의 최소 상태 즉시 모드 그래픽 사용자 인터페이스입니다. Nuklear는 애플리케이션을 위한 단순한 비돌출형 그래픽 사용자 인터페이스이고 어떠한 종속성도 가지고 있지 않는, 기본 렌더 백엔드 혹은 OS 창 및 입력 핸들러이지만 매우 규격화된 라이브러리를 제공하기보다는 단순한 입력 상태를 통해 접근하고 출력으로써 원시적인 도형을 그리는 명령을 그립니다. 많은 수의 플랫폼을 추상화하고 백엔드를 렌더링하는 것을 시도하는 계층화된 라이브러리보다 사실상의 UI에만 집중합니다.

  2. 하이라이트

    • 그래픽 사용자 인터페이스 툴킷
    • 단일 헤더 라이브러리
    • C89(a.k.a. ANSI C 혹은 ISO C90)로 작성됨
    • 작은 코드베이스 (~1.8만 줄)
    • 휴대성, 효율성 및 단순성에 중점을 둠
    • 종속되지 않음 (필요없다면 기본 라이브러리마저)
    • 완전 사용자화 및 스킨 씌우기 가능
    • 필요 시 전면적 메모리 컨트롤로 적은 메모리 발자국을 남김
    • UTF-8 지원
    • 전역 및 숨겨진 상태 없음
    • 사용자화 가능한 라이브러리 모듈 (필요한 것만 컴파일하고 사용 가능)
    • 선택적 폰트 굽기 및 정점 버퍼 출력
    • GitHub에서 코드 사용 가능

  3. 기능

    • 플랫폼 종속 코드가 절대대 아님
    • 메모리 관리 조종 범위:
      • 표준 라이브러리에서 모든 것을 할당함으로써의 쉬운 사용에서
      • 라이브러리 내의 모든 바이트를 조종하기까지
    • 폰트 핸들링 조종 범위:
      • 모든 것에 자신만의 폰트를 사용하는 것부터
      • 이 라이브러리의 폰트 베이킹과 핸들링 API를 사용하는 것까지
    • 그림 출력 제어 범위:
      • 그림 호환성을 가진 고수준 API용 간단한 도형에서
      • 하드웨어 접근 가능한 안티앨리어싱이 적용된 정점 버퍼 출력까지
    • 사용자 지정 가능한 색상과 속성의 범위:
      • 간단한 색상 테이블을 채움으로써의 간단한 색상 변화에서
      • 위젯을 꾸미기 위한 스키닝의 사용 가능성을 포함한 완전한 조종까지
    • 다음 위젯 범위를 포함한 융통성 있는 UI:
      • 버튼, 체크박스, 슬라이더 등의 기본 위젯부터
      • 추상적인 콤보박스, 컨텍스트 메뉴 등의 고급 위젯까지
    • 필요한 것만을 컴파일하기 위한 컴파일 시간 설정
      • 표준 라이브러리를 사용하거나 링크하기 싫을 때 사용할 수 있는 서브셋
    • 프레임마다 업데이트하는 대신 유저의 입력이 들어올 때마다 업데이트하기 위해 쉽게 수정할 수 있음

  4. 사용법

    5. Nuklear 라이브러리는 하나의 단일 헤더에 자체 포함되어 있고 헤더 전용 모드나 실행 모드에서 사용할 수 있습니다. 헤더 모드는 포함되었을 때 기본으로 사용되고 이 헤더를 다른 헤더에 포함하는 것을 허용해 실제 실행을 포함하지 않습니다.

    실행 모드는 하나의 C/CPP 파일에 포함하기 전에 전처리 메크로 NK_IMPLEMENTATION을 선언언하는 것을 요구합니다.

    예:

    #define NK_IMPLEMENTATION #include "nuklear.h"

    또한 추가적으로 "추가 선언" 섹션에 열거되어 있는 기호를 실행 모드에서 헤더 밑에 추가적인 기능성을 사용하고 싶거나 라이브러리에 대한 더욱 많은 제어를 필요로 할 때 선언할 수 있습니다.

    ⚠️
    Nuklear가 포함될 때 같은 컴파일러 플래그를 선언하십시오. 이 동작은 매우 중요하며, 하지 않는다면 컴파일러 오류, 혹은 훨씬 나쁘게 스택 손상으로 이어질 수 있습니다.

  5. 플래그

    6. 플래그 설명
    NK_PRIVATE 선언된다면 모든 함수를 정적 함수로 선언해, 실행하는 파일의 내부에서만 접근 가능하게 합니다.
    NK_INCLUDE_FIXED_TYPES 선언된다면 헤더 <stdint.h>를 포함할 것이며 그렇지 않은 경우에는 Nuklear가 올바른 형을 선택할 것입니다. 이마저 실패할 경우에는 컴파일 오류를 발생시킬 것이며 맞는 형을 선택해야 합니다.
    NK_INCLUDE_DEFAULT_ALLOCATOR 선언된다면 헤더 <stdlib.h>를 포함하고 로딩하는 파일에 맞추어 추가적인 기능을 제공할 것입니다.
    NK_INCLUDE_STANDARD_IO 선언된다면 헤더 <stdio.h>를 포함하고 로딩하는 파일에 맞추어 추가적인 기능을 제공할 것입니다.
    NK_INCLUDE_STANDARD_VARARGS 선언된다면 헤더 <stdarg.h>를 포함하고 로딩하는 파일에 맞추어 추가적인 기능을 제공할 것입니다.
    NK_INCLUDE_STANDARD_BOOL 선언된다면 nk_bool을 헤더 <stdbool.h>에 포함시킬 것이고 그렇지 않는다면 Nuklear는 nk_bool을 int 자료형으로 선언합니다.
    NK_INCLUDE_VERTEX_BUFFER_OUTPUT 이를 선언하면 이 라이브러리에 정점 그리기 명령 리스트 백엔드를 추가합니다. 이는 대기열 명령을 정점 그리기 명령으로 변환하는 것을 가능하게 합니다. 이는 OpenGL, DirectX, Vulkan, Metal 등에 대한 하드웨어 접근 형식이 필요할 때 주로 사용됩니다.
    NK_INCLUDE_FONT_BAKING 이를 선언하면 이 라이브러리에 stb_truetype, stb_rect_pack 구현을 추가하고 폰트의 베이킹과 렌더링을 제공합니다. 만약 이 폰트 핸들러를 사용하고 싶지 않거나 폰트 핸들러가 이미 있다면 이를 선언하지 않아도 됩니다.
    NK_INCLUDE_DEFAULT_FONT 이를 선언하면 기본 폰트 ProggyClean.ttf를 라이브러리에 추가합니다. 이 폰트는 폰트 아틀라스로 로드될 수 있고 트루타입 폰트 없이도 이 라이브러리를 사용할 수 있게 해 줍니다.
